# Task Tracker

Веб-додаток для управління списком завдань з авторизацією, розділенням серверної/клієнтської логіки та real-time повідомленнями.

## Стек технологій
* **Framework:** Next.js 16 (App Router)
* **Auth:** Auth.js (NextAuth) + Google Provider
* **UI Library:** MUI (Material UI)
* **Forms:** React Hook Form + Zod (валідація)
* **Data:** MockAPI.io (зовнішній сервіс)
* **Language:** TypeScript

## Інструкція з розгортання

### Локально:
1. Клонуйте репозиторій.
2. Встановіть залежності: `npm install`.
3. Створіть файл `.env` у корені проекту та додайте змінні (див. розділ нижче).
4. Запустіть додаток: `npm run dev`.
5. Відкрийте [http://localhost:3000](http://localhost:3000).

### На Vercel:
1. Завантажте код на GitHub/GitLab.
2. Підключіть репозиторій до Vercel.
3. Додайте змінні оточення у налаштуваннях проекту.
4. Натисніть "Deploy".

## Налаштування Google OAuth (.env)

Для роботи авторизації необхідно створити проект у [Google Cloud Console](https://console.cloud.google.com/), налаштувати "OAuth consent screen" та створити "OAuth 2.0 Client IDs".

Потрібні змінні:
```env
GOOGLE_CLIENT_ID=ваш_client_id
GOOGLE_CLIENT_SECRET=ваш_client_secret
NEXTAUTH_SECRET=випадковий_рядок_для_шифрування
NEXTAUTH_URL=http://localhost:3000
MOCKAPI_URL=https://68e9272ef2707e6128cdd00e.mockapi.io/tasks
```

## Архітектура проекту

У проекті використано комбінований підхід для демонстрації різних можливостей Next.js:

1. **Server Actions:**
   * Використовуються для **створення завдань** та **зміни статусу** (виконано/не виконано).
   * Це забезпечує тісну інтеграцію з серверним станом та дозволяє використовувати `revalidatePath` для миттєвого оновлення UI без додаткових клієнтських запитів.
   * Валідація Zod виконується як на клієнті (для UX), так і на сервері (для безпеки).

2. **Proxy (Middleware):**
   * Використовує файл `src/proxy.ts` (згідно зі стандартами Next.js 16) для захисту внутрішніх роутів.
   * Неавторизовані користувачі автоматично перенаправляються на сторінку `/login`.

3. **API Routes (Route Handlers):**
   * **Видалення завдань (`DELETE /api/tasks/[id]`):** Реалізовано через API, щоб продемонструвати вміння працювати зі стандартними HTTP методами та клієнтським `fetch`.
   * **Оновлення списку (`GET /api/tasks`):** Використовується клієнтською частиною (кнопка Refresh) для отримання актуальних даних без повного перезавантаження сторінки.
   * **SSE (`GET /api/events`):** Реалізація Server-Sent Events для real-time повідомлень.

## Структура папок
* `src/app` — роутинг та сторінки (App Router).
* `src/actions` — серверні дії (Server Actions).
* `src/components` — UI компоненти (MUI).
* `src/lib` — логіка, валідація, робота з даними.
* `src/api` — (всередині app) Route Handlers.
